Known Bugs

Some things cant be fixed or avoided in modding a game with a closed source code. If you experience wonky gameplay and general oddities have a look here, maybe its a known and already reported problem. There are usually workarounds available for you to fix it.

If you cant find the source of your problems on the list, please come by bay12 and share. In almost all cases it is a typo or simple error in the raws that can be fixed in a future release.

Workshop reactions are red, even with all the materials available.
Bug: If a workshop is in a burrow, even if the burrow is inactive, then it accepts only materials within the same burrow.
Solution: Make a bigger burrow including the materials, or keep the material stockpiles within the same burrow. Or delete the burrow.



I have many weird boulders lying around, named sparks, knowledge, etc...
Bug: Many things in the mod depend on temperature. Things that should melt or boil away don't without it.
Solution: Open the GUI or Init.txt and set Temperature to ON. This can be done in a running fortress, you only have to restart DF. You do not need to gen a new world. You can also turn off "smoke effects" in the misc features section. This will decrease the amount of these boulders significantly.



Worldgen crashes.
Bug: Dwarf Fortress crashes to desktop while creating a world.
Solution: I have never been able to replicate this. It appears to be at random, with major differences in the PC system people use. Try shorter histories, <100 years. If the problem persists, please leave a detailed report in the forum. Please mention the mod version, tileset used, if settings have been changed, and how much virtual memory/ram you have.



I have an error log.
Bug: The Dwarf Fortress main folder shows a errorlog.txt. This is created when certain entry in the raws are missing. In 90% of the cases this is completely harmless. It is caused by enabling some settings, and disabling other.
For example you remove warpstone. The error log will tell you that warpstone is missing in the reactions for warpstone landmines and turrets.
Solution: The default setting has no error log. If your setting has a very big error log, please leave me a message in the forum. Otherwise just ignore it, it will do no harm.
