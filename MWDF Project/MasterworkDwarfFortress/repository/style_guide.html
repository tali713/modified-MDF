Headings

Headings take the form of primarily <h1> tags, but if for some reason you feel the need for subheadings in your page, <h2> tags are acceptable. You should avoid using <h3> and beyond, since they're not formatted for by the CSS and you honestly shouldn't have any reason to use them.

H1 headings are, by default, formatted to be centered on the page, bold, and are nice and big. H2 headings are left-aligned, and slightly smaller.

Notably, headings should almost always have an id, corresponding to an internal link in the page index up top.

If you want a more in-depth look at the basic header for files, open this document in a text-editor as opposed to a web browser. There's a blank one in comments at the bottom of this webpage.

That should be all for headers for now

Body Text

Body text is, as always, one of the more complex parts of an HTML page. Not only for its content, which is of course important, but its formatting. Keeping in mind how HTML works will be important if you want to use certain characters, for example, < cannot be directly written into an HTML page. It has to be represented by &lt;, and yes, I just had to write &amp;lt; to write that. You may want to avoid certain kinds of punctuation altogether with HTML.

Plaintext can be formatted in several ways with HTML, of course. Everything is in <p> tags, of course, but you can use <span> tags to select small sections of text to style via inline CSS, if you're daring, or, if you prefer to just edit in HTML, use any of the following tags.

<strong> ::: <em> ::: <s> ::: <sub> ::: <sup>

For the most part you probably won't need to use these all that much; just be aware that <strong> should always be used instead of <b>, and <em> should always be used instead of <i>. They do the same thing visually, but it's just good practice. Most of your formatting should be in the form of page layout, though.

General Info

General Info should introduce the player to the concept of the race, but should not provide explicit advice on how to play. It should be purely informative. Topics should be organized in the following order:
Race
The race itself should be described, with any anatomical quirks disclosed, such as the fact that succubi bleed fire, and kobolds are small and carnivorous. Basic personalities should also be glanced over here, such as orcs favoring warfare and traditions. Most of the current ones are okay as-is, content wise. If it's weird, then it should go here. The race description should be made in the form of paragraphs, and should be headered merely by the name of the race, plural.
Castes
Castes that do anything other than set gender should be assembled into a list, with information about the attribute and skill modifiers of the caste and a description of what they are included. If they are acquired by some nonstandard means, the method of acquisition should be noted as well.
Nobles
Nobles should be listed, of course, and their duties made clear. Comparing them to vanilla nobles is perfectly okay
Basic Necessities
General gameplay quirks should be noted here. Kobolds require a source of meat, most succubi-specific workshops require lava beneath them, etc. Also, note "optional" needs too; dwarves need alcohol, orcs practically need to practice martial skills, things that average members of the race would rather die than not have, really.
Other
If there's something else, put it after everything else on the page.

Getting Started

Getting Started should be a basic guide to the race for players, both new and old. You can largely assume proficiency with vanilla features, but mod features should never be assumed. Topics should be organized in the following order:
World Generation (optional)
Any notes as to how world gen might affect game difficulty should go here, if any. Orcs are likely to eventually care a lot about worldgen once someone figures out a way to permit/unpermit reactions based on what races actually exist and are nearby, as are humans. Invaders are also important, of course, especially for succubi and kobolds. Things like that.
Before Embarking
What should the player consider before embarking; basic equipment, skills, and location. Playing kobolds on a glacier, for example, is typically suicidal. Playing kobolds and taking 0 animals or meat is also bad.
Early Game
Basic industries can be described here, but an emphasis should be made on ones required for survival. Food, drink, and setting up potential trade goods or low-grade military equipment.
Midgame
This section is for after a fortress is sustainable, after food and basic furniture is available in decent amounts. How to acquire decent military equipment and get your citizens happy should be emphasized here.
Late Game
Capstone workshops and industries should be described here. Workshops with significant requirements to build, or industries with long production chains belong in this section.

Buildings, Items, Materials & Pets

All of these should list their contents and provide basic descriptions of them. Buildings should have each workshop prepended by an image of the workshop, and race-specific pages should only include workshops that have race-specific reactions or that are wholly race-specific; multi-race buildings and reactions should not be described on race pages, and such should instead be placed in a dedicated multi-race page.
